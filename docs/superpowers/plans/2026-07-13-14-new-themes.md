# 14 New CSS Themes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Thêm 14 CSS theme class vào `src/landing/themes.css` để mỗi trong 20 version landing page có một visual identity riêng biệt.

**Architecture:** Tất cả 14 theme được append vào `src/landing/themes.css` theo cùng token structure với 6 theme hiện có. Không có logic nào thay đổi — chỉ là CSS data. Mapping theme → recipe (version) là out of scope của plan này và sẽ làm riêng sau.

**Tech Stack:** CSS custom properties (CSS variables), Next.js 15, `src/landing/themes.css`

**Spec:** `docs/superpowers/specs/2026-07-13-14-new-themes-design.md`

---

## File Structure

| File | Action | Mô tả |
|---|---|---|
| `src/landing/themes.css` | **Modify** | Append 14 theme class block, mỗi block ~25 CSS vars |

Không tạo file mới. Không sửa recipe, variant, hay bất kỳ file TypeScript nào.

**Files:** không thay đổi

```bash

```

Mở `http://localhost:3000/versions` trong browser.

- [ ] **Step 2: Kiểm tra từng theme mới hiển thị đúng màu**

Tại `/versions`, page hiển thị tất cả version với theme tương ứng. Với mỗi theme mới, kiểm tra:
- Background hero section đúng màu theo spec
- Accent color (button, highlight) đúng
- Không có màu trắng/đen kỳ lạ do thiếu CSS var

Nếu một theme chưa được gán vào recipe nào, nó sẽ không xuất hiện trên `/versions`. Đó là bình thường — scope này chỉ thêm CSS class, mapping vào recipe là task tiếp theo ngoài scope.

- [ ] **Step 3: Chạy no-emoji check**

```bash
python3 scripts/check_no_emoji.py
```

Expected: pass (không có emoji trong CSS)

- [ ] **Step 4: Chạy contrast check**

```bash
python3 scripts/contrast.py
```

Nếu có theme fail contrast (ratio < 4.5:1 cho text), điều chỉnh `--lp-primary` của theme đó sang màu tối hơn.

Các dark theme (charcoal, forest, crimson) thường pass vì text là light-on-dark. Các pastel theme cần `--lp-primary` đủ tối.

- [ ] **Step 5: Commit nếu có fix contrast**

Nếu bước 4 yêu cầu điều chỉnh màu:

```bash
git add src/landing/themes.css
git commit -m "fix(theme): adjust primary contrast for WCAG AA compliance"
```
